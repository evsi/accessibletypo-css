Minimal accessible typography

With this repository I am trying to create a CSS file which should help to validate a simple HTML document against the AAA criteria of the WCAG. Moreover it should provide styles which make a document easy to read for everyone

This documentation itself should not violate any Guidelines provided by the WCAG

Motivation and decision making

  • The CSS file should provide the means to meet the Success Criteria for the guideline 1.4 - Distinguishable of the WCAG 2.0 on an AAA level.
  • The CSS file should be as small as possible.
  • There should be no need to add many classes to the HTML document.

Success Criteria and with the according Tests

The success criteria for the guideline 1.4 contain criteria about audio. As this is a pure text document this criteria ar left beside!

Visual Presentation 1.4.8 AAA

For details see the quick reference to Web Content Accessibility Guidelines (WCAG).

The following list describes the points which should be fullfilled to meet the success criteria,

  1. Foreground and background colors can be selected by the user.

    This criterion is met by using : G148: Not specifying background color, not specifying text color, and not using technology features that change those defaults

  2. Width is no more than 80 characters or glyphs (40 if CJK).

    This criterion is met by using : C20: Using relative measurements to set column widths so that lines can average 80 characters or less when the browser is resized

    The following CSS rules are introduced to fullfill this requirement

      html {
    font-size: 100%;
    font-family: 'Open Sans', Verdana, 'Lucida Grande', sans-serif;
    line-height: 1.5; 
  }

    Set the font size for the hole document to 100% which would be for most browser 16px.

      .main-content { /*this class should ensure that a body of text normaly a p element has lines no longer then 80 signs*/
    max-width: 36rem;
    margin-right: auto;
    margin-left: auto;
    padding-right: 1rem;
    padding-left: 1rem;
  }

    Use a class like "main-content" on the encapsulating module of your content. in combination with the font "Open Sans" and the font "Verdana" this should produce lines which are no longer then 80 signs

  3. Text is not justified (aligned to both the left and the right margins).

  4. Line spacing (leading) is at least space-and-a-half within paragraphs, and paragraph spacing is at least 1.5 times larger than the line spacing.

  5. Text can be resized without assistive technology up to 200 percent in a way that does not require the user to scroll horizontally to read a line of text on a full-screen window.

Meeting SC 1.4.4 Resize text

Meeting this criterion with the help of G146: Using liquid layout.

  1. Zoom content to 200%
  2. Check whether all content and functionality is available with no horizontal scrolling.

Up to a certain amount of zooming text is zooming with its container elements when the viewport gets too small there will be text wrapping. At a certain stage of zooming words get too long for the viewport, at this moment the browser provides scrollbars.

Meeting SC 1.4.6 Contrast (Enhanced)

Meeting this criterion with the help of G148: Not specifying background color, not specifying text color, and not using technology features that change those defaults.

  1. Look in all places that text color can be specified
  2. Check that text color is not specified
  3. Look in all areas that background color or image used as a background can be specified
  4. Check that no background color or image used as a background is specified
